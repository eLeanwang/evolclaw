## 测试覆盖的功能点

### 核心功能
1. ✅ 消息缓存机制
   - 内存缓存，按 sessionId 组织
   - 缓存上限 100 条消息
   - 自动清理过期消息（1小时）

2. ✅ 项目切换时的消息路由
   - 活跃项目：直接发送
   - 非活跃项目：缓存消息
   - 使用 AsyncLocalStorage 传递上下文

3. ✅ Interrupt 机制优化
   - 同项目：触发中断
   - 不同项目：不触发中断
   - 避免误中断原项目任务

4. ✅ 文件处理
   - 文件存在：正常发送
   - 文件不存在：跳过发送，提示用户

5. ✅ 多聊天独立性
   - 每个聊天的缓存独立
   - 互不干扰

### 边界情况
1. ✅ 无上下文时的消息发送（命令响应）
2. ✅ 无效 sessionId 格式的处理
3. ✅ 并发上下文的支持
4. ✅ 错误处理和恢复
5. ✅ 队列长度和处理状态报告
