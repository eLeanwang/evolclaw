# Bug 修复总结 (2026-03-10)

## 发现的严重 Bug

### 1. 测试删除生产数据目录 ⚠️ CRITICAL

**文件**: `tests/unit/database.test.ts`

**问题**:
```typescript
afterEach(() => {
  db.close();
  rmSync('./data', { recursive: true, force: true }); // 删除整个 data 目录！
});
```

**影响**:
- 每次运行 `npm test` 都会删除整个 `data/` 目录
- 丢失所有配置文件：config.json, config.json.backup, config.json.template
- 丢失生产数据库：sessions.db
- 丢失文档：README.md
- 导致服务无法启动

**修复**:
```typescript
const testDataDir = './data/test-db';
afterEach(() => {
  db.close();
  if (existsSync(testDataDir)) {
    rmSync(testDataDir, { recursive: true, force: true }); // 只删除测试目录
  }
});
```

**提交**: c9e9dbb

---

### 2. 消息重复发送

**文件**: `src/core/message-processor.ts`

**问题**:
- `flusher.flush()` 发送一次内容
- 之后又从 `getFinalText()` 获取完整文本再发送一次
- 导致每条消息被发送两次

**修复**:
- 在 flush 之前处理文件标记
- 使用 `stripFromBuffer()` 移除标记
- 只调用一次 `flush()`

**提交**: e3833c9

---

### 3. 工具调用后文本响应消失

**文件**: `src/core/message-processor.ts`

**问题**:
```typescript
if (event.type === 'result' && event.result) {
  if (!flusher.hasSentContent()) {  // 这个检查导致文本被跳过
    flusher.addText(event.result);
  }
}
```

**原因**:
1. 工具调用触发 `addActivity()` → `scheduleFlush()`
2. 3秒后自动 flush，发送工具列表
3. `hasSentContent()` 变为 `true`
4. 最终文本因为条件检查被跳过

**修复**:
```typescript
if (event.type === 'result' && event.result) {
  flusher.addText(event.result);  // 移除条件检查
}
```

**提交**: 007516e

1. **测试隔离**: 所有测试必须使用独立的测试目录/文件
2. **路径检查**: 删除操作前检查路径，避免误删生产数据
3. **代码审查**: 重点检查 `rmSync`, `unlinkSync`, `rmdirSync` 等危险操作
4. **配置备份**: 定期备份 `data/config.json` 到安全位置
